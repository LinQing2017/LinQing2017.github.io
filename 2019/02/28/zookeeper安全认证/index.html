ZooKeeper Access Control

Zookeeper官方文档有针对ACLs章节的相关讨论，下面简单终结。

Zookeeper 的权限包括：CREATE、READ、WRITE、DELETE、ADMIN。

Zookeeper 的身份的认证方式：

认证方式 说明
world 表示所有客户端都能够通过认证
auth 表示任何经过身份验证的用户，当使用Kerberos时表示对应的principal账户通过认证
digest 用户名/密码方式通过认证
host 客户端主机名通过认证
ip 客户端IP通过认证
